# LocxOS 调试与学习手册

本文档面向 Windows + WSL 环境，给出可直接执行的构建、运行、串口观察和 GDB 调试流程。

## 1. 环境准备

建议固定使用同一个 WSL 发行版，例如 Ubuntu-24.04。

### 1.1 查看与设置默认发行版（可选）

```powershell
wsl -l -v
wsl --set-default Ubuntu-24.04
```

### 1.2 安装依赖（在 Ubuntu-24.04）

```powershell
wsl -d Ubuntu-24.04 bash -lc "sudo apt update; sudo apt install -y build-essential grub-common grub-pc-bin xorriso qemu-system-x86 gdb"
```

## 2. 构建与启动

说明：当前项目支持使用系统 GCC 路径，命令中通过 `CROSS=` 明确启用。

### 2.1 清理并构建

```powershell
wsl -d Ubuntu-24.04 bash -lc "cd /mnt/c/base/LocxOS; make clean; make all CROSS="
```

### 2.2 生成 ISO

```powershell
wsl -d Ubuntu-24.04 bash -lc "cd /mnt/c/base/LocxOS; make iso CROSS="
```

### 2.3 图形模式运行（VGA）

```powershell
wsl -d Ubuntu-24.04 bash -lc "cd /mnt/c/base/LocxOS; make run CROSS="
```

预期现象：

- QEMU 窗口显示 `LocxOS boot ok`。

## 3. 串口日志调试

串口模式会把内核日志直接打印在终端，适合早期调试。

```powershell
wsl -d Ubuntu-24.04 bash -lc "cd /mnt/c/base/LocxOS; make run-serial CROSS="
```

预期输出：

- `[serial] LocxOS booting...`
- `[serial] VGA message rendered`

## 4. GDB 断点调试

### 4.1 终端 A：启动 QEMU 并等待调试器

```powershell
wsl -d Ubuntu-24.04 bash -lc "cd /mnt/c/base/LocxOS; make debug CROSS="
```

### 4.2 终端 B：附加 GDB（快捷方式）

```powershell
wsl -d Ubuntu-24.04 bash -lc "cd /mnt/c/base/LocxOS; make gdb CROSS="
```

### 4.3 手动 GDB（可选）

```bash
gdb build/kernel.elf
(gdb) target remote :1234
(gdb) break kmain
(gdb) continue
```

## 5. 常见问题排查

### 5.1 `grub-mkrescue: No such file or directory`

原因：未安装 `grub-common` 或环境不是预期发行版。

处理：

1. 确认发行版：`wsl -l -v`
2. 显式执行：`wsl -d Ubuntu-24.04 ...`
3. 安装依赖：见本文 1.2

### 5.2 链接时报 PIE 相关错误

原因：系统 GCC 默认启用 PIE。

处理：

- 项目 Makefile 已加 `-fno-pie` 和 `-no-pie`。
- 若你修改了编译参数，请保留这两个选项。

### 5.3 QEMU 窗口无输出或闪退

建议顺序：

1. 先执行 `make run-serial CROSS=` 看终端日志。
2. 再用 `make debug CROSS=` + `make gdb CROSS=` 断到 `kmain`。
3. 检查是否使用了正确的 ISO：`build/LocxOS.iso`。

## 6. 学习路线建议

1. 第一步：阅读 `boot.s` 与 `long_mode_init.s`，理解 CPU 模式切换。
2. 第二步：阅读 `paging.s`，理解恒等映射和 2 MiB 页。
3. 第三步：阅读 `kernel.c`、`vga.c`、`serial.c`，建立最小内核运行直觉。
4. 第四步：扩展 IDT/异常处理，让系统具备可诊断能力。

## 7. 后续扩展建议

1. 增加 IDT 与异常向量打印。
2. 增加物理内存探测和简单页分配器。
3. 增加基础字符串与格式化输出并接入串口/VGA。
