# 涉及的汇编语言基础

对于初学者来说，了解本项目是如何进入 C 语言环境的，必然需要接触到最底层的汇编语言（Assembly）。在 kernel/arch/x86_64/ 目录下，包含几个关键的 .s（汇编代码）文件：boot.s、long_mode_init.s 和 paging.s。

本篇文档将介绍本项目中涉及的基础汇编语法和概念。

## 1. GNU 汇编器（GAS）语法简介

我们使用的是 GNU 汇编器（通过 gcc 间接调度），采用 **AT&T 语法格式**，它与我们在很多国内教材上学的 Intel 格式有所不同：

*   **操作数顺序**：指令 源操作数, 目的操作数。例如 mov %eax, %ebx 表示把 eax 的值拷贝给 ebx。
*   **寄存器前缀**：所有寄存器名称前面都要加 %，比如 %eax、%rsp。
*   **立即数前缀**：所有常数（立即数）前面都要带 $，比如 $0x10、$512。

## 2. 核心伪指令 (Directives)

以 . 开头的单词是**伪指令**，它们不是真正的机器 CPU 执行的指令，而是告诉“汇编器”或“链接器”该怎么做：

*   .section <段名>：定义或切换代码或数据的存放区域。例如：
    *   .section .multiboot：存放给 GRUB 识别的多重引导头。
    *   .section .text：存放可执行的代码。
    *   .section .bss：存放未初始化或初始化为0的数据（比如我们的调用栈、页表占用的空间）。
*   .global <符号> 或 .extern <符号>：声明一个符号（函数或变量）。.global 告诉链接器这是供全局使用的模块入口（如 setup_page_tables）；.extern 则声明这个符号定义在别的文件里（例如在 C 代码里的 kmain）。
*   .align <字节数>：指示汇编器对齐地址。如 .align 4096 要求接下来的数据从 4096 的整数倍地址开始存放（4K，正是一个内存页的大小）。硬件分页机制严格要求页表基址对齐。
*   .long <值> / .short <值> / .quad <值>：分别在当前位置直接写入 32 位、16 位、64 位的常数值。我们在写 Multiboot2 头和 GDT（全局描述符表）时经常用到。
*   .skip <字节数>：分配一块指定大小并且填满零的空间。比如用来做堆栈和页表的内存 .skip 4096。
*   .code32 与 .code64：指示汇编器分别按照 32 位和 64 位体系结构来编译下面的指令代码。由于从 BIOS/GRUB 接手控制权时 CPU 依然处于 32 位保护模式，必须先用 32 位指令完成配置后再跳转进入真正的 64 位模式（长模式）。

## 3. 重要寄存器与指令

在内核引导阶段，我们主要做三件事：**设置栈**、**配置页表**、**切换长模式（Long Mode）**。主要涉及以下寄存器和指令：

### 寄存器

*   **通用寄存器**：%eax, %ebx, %ecx, %edx（32位）；%rax, %rbp, %rsp（64位）。主要用于临时存储数据、指针或传参。特别注意 %esp / %rsp 分别是 32/64 位栈顶指针寄存器。
*   **控制寄存器 (CR0, CR3, CR4)**：
    *   **CR3**：存放最高级页表的物理基地址。
    *   **CR4**：设置页面扩展功能，比如我们在 long_mode_init.s 里打开的 PAE（物理地址扩展，bit 5）。
    *   **CR0**：这是系统的“总开关”。通过将 PG 位（bit 31）置 1，正式开启分页；PE 位（bit 0）控制是否开启保护模式。
*   **MSR (模型特定寄存器)**：如 EFER（扩展特征启用寄存器）。我们要让 CPU 知道我们打算玩 64 位系统，就必须通过 
dmsr 和 wrmsr 指令，设置 EFER 寄存器的 LME 位（长模式启用）。

### 常见指令

*   mov , %eax：将数值 0x83 (二进制 10000011) 放进 eax。
*   or , %eax：按位或运算，常用于在一个原有的寄存器里叠加新的“标志位”（如打开标志 RW 和 P）。
*   shl , %eax：逻辑左移，相当于乘以 2 的次方。我们在构建页表基址时用于计算地址。
*   lgdt <地址>：加载新的全局描述符表（GDT）地址，它定义了各个段的安全特权和位宽属性。
*   ljmp , ：远跳转指令。通过更换代码段为 0x08（新 GDT 里我们刚写好的 64 位描述符）同时跳向 64 位的汇编入口代码，从而正式切到 64 位状态。
*   call <函数名> / 
et：调用函数和从函数返回。
*   cli：关中断（Clear Interrupt Flag）。因为系统刚启动还什么都没准备好，接收到外部硬件中断绝对会崩溃，所以在最初我们要把它关掉。
*   hlt：暂停 CPU 运行直到下一个中断到来。通常被放在末尾组成 jmp halt 死循环，防止万一 C 语言的 main 函数跳出了，CPU 不会跑飞执行乱码。

通过对上述概念的熟悉，读者应当能够读懂这几个汇编文件是如何相互协作、一步步初始化硬件最后引出 C 语言 kmain 入口函数的。
