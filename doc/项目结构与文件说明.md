# LocxOS 项目结构与文件说明

本文档用于说明当前项目目录结构、关键文件职责、启动路径和后续可扩展方向，方便学习和维护。

## 1. 项目目标（当前阶段）

当前实现的是一个最小可运行的 x86_64 内核原型：

1. GRUB2 使用 Multiboot2 加载内核 ELF。
2. 汇编代码建立最小页表并切换到 Long Mode。
3. 跳转到 C 语言入口 `kmain`。
4. VGA 显示启动信息，串口打印调试日志。
5. 内核进入 `hlt` 循环。

## 3. 启动链路（从开机到 C 入口）

### 3.1 引导阶段

1. GRUB 读取 `iso/boot/grub/grub.cfg`。
2. `multiboot2 /boot/kernel.elf` 将内核加载到内存。
3. 跳转到链接脚本指定入口 `_start`（定义在 `boot.s`）。

### 3.2 保护模式到长模式

在 `boot.s` 中：

1. 关闭中断并设置临时栈。
2. 调用 `setup_page_tables`（`paging.s`）初始化 P4/PDPT/PD。
3. 调用 `enable_long_mode`（`long_mode_init.s`）设置寄存器并远跳转到 64 位代码。

在 `long_mode_init.s` 中：

1. 写入 CR3 指向 P4。
2. 开启 CR4.PAE。
3. 设置 EFER.LME。
4. 开启 CR0.PG（同时保留 PE）。
5. 加载最小 GDT 并执行远跳转进入 `.code64`。

### 3.3 进入 C 内核

1. 64 位入口设置数据段寄存器和 RSP。
2. 调用 `kmain`（`kernel/kernel.c`）。
3. `kmain` 进行串口初始化、VGA 文本输出，然后进入 `hlt` 循环。

## 4. 关键文件职责

## 4.1 构建系统

- `Makefile`
  - 定义编译参数（freestanding、禁用 PIE、64 位）。
  - 生成 `build/kernel.elf`。
  - 打包 `build/LocxOS.iso`。
  - 提供 `run`、`run-serial`、`debug`、`gdb` 目标。

- `kernel/arch/x86_64/linker.ld`
  - 指定 ELF 入口 `_start`。
  - 指定段布局（`.multiboot`、`.text`、`.rodata`、`.data`、`.bss`）。
  - 当前加载基址为 1 MiB。

## 4.2 启动与架构代码

- `kernel/arch/x86_64/boot.s`
  - 放置 Multiboot2 Header。
  - 作为 32 位初始入口，设置栈并调用页表与长模式初始化。

- `kernel/arch/x86_64/paging.s`
  - 建立最小页表结构。
  - 当前使用 2 MiB 大页恒等映射低端 1 GiB，满足最小启动需求。

- `kernel/arch/x86_64/long_mode_init.s`
  - 执行进入 Long Mode 的关键寄存器配置。
  - 提供 64 位入口 `long_mode_entry` 并调用 C 内核。

## 4.3 内核功能代码

- `kernel/kernel.c`
  - 内核主入口 `kmain`。
  - 当前功能：串口日志 + VGA 输出 + 停机循环。

- `kernel/arch/x86_64/vga.c`
  - 文本模式显存写入（地址 `0xB8000`）。
  - 提供清屏与字符串输出。

- `kernel/arch/x86_64/serial.c`
  - 初始化 COM1（`0x3F8`）串口。
  - 提供阻塞式字符串发送，便于早期调试。

## 4.4 预留文件说明

- `common/printf.c`、`common/string.c`
  - 当前未接入构建，后续可扩展通用库。

- `kernel/arch/x86_64/gdt.c`、`kernel/arch/x86_64/idt.c`
  - 当前未启用；后续可迁移 GDT/IDT 初始化逻辑到 C。

## 5. 当前可观察行为

1. 屏幕显示：`LocxOS boot ok`。
2. 串口输出：
   - `[serial] LocxOS booting...`
   - `[serial] VGA message rendered`
3. 系统保持在 `hlt` 循环等待中断。

## 6. 推荐后续学习顺序

1. 先理解 `boot.s` + `paging.s` + `long_mode_init.s` 的模式切换。
2. 再阅读 `linker.ld`，理解入口和段布局。
3. 然后看 `kernel.c`、`vga.c`、`serial.c` 的最小功能闭环。
4. 最后补全 IDT 与异常处理，建立“可诊断”的内核基础设施。
