# LocxOS 代码对照学习索引

本文档用于把“源码文件、关键符号、运行时现象、调试断点”串成一条学习路径。

## 1. 启动时序总览

1. 固件把控制权交给 GRUB。
2. GRUB 读取启动配置并加载内核 ELF。
3. CPU 进入内核入口 _start（32 位代码）。
4. 初始化页表，开启 long mode。
5. 远跳转进入 64 位入口 long_mode_entry。
6. 调用 C 入口 kmain。
7. 输出串口日志和 VGA 文本，随后进入 hlt 循环。

## 2. 文件到功能的映射

## 2.1 引导与布局

- [iso/boot/grub/grub.cfg](../iso/boot/grub/grub.cfg)
  - 作用：定义 GRUB 菜单项，使用 multiboot2 加载内核。
  - 学习点：引导器如何找到内核镜像。

- [kernel/arch/x86_64/linker.ld](../kernel/arch/x86_64/linker.ld)
  - 作用：定义内核入口与段布局。
  - 学习点：为什么把镜像放在 1MiB，为什么要 KEEP Multiboot 节。

- [kernel/arch/x86_64/boot.s](../kernel/arch/x86_64/boot.s)
  - 关键符号：multiboot_header、_start、stack_top。
  - 作用：32 位初始入口，设置栈并进入模式切换流程。
  - 学习点：Multiboot2 头部校验、早期 cli 的意义。

## 2.2 模式切换与分页

- [kernel/arch/x86_64/paging.s](../kernel/arch/x86_64/paging.s)
  - 关键符号：setup_page_tables、p4_table、pdpt_table、pd_table。
  - 作用：构建最小页表，恒等映射低端 1GiB。
  - 学习点：2MiB 大页、页表项标志位 P/RW/PS。

- [kernel/arch/x86_64/long_mode_init.s](../kernel/arch/x86_64/long_mode_init.s)
  - 关键符号：enable_long_mode、long_mode_entry。
  - 作用：配置 CR3/CR4/EFER/CR0，切入 64 位。
  - 学习点：PAE、LME、PG 的先后关系与必要性。

## 2.3 C 内核入口与输出

- [kernel/kernel.c](../kernel/kernel.c)
  - 关键符号：kmain。
  - 作用：初始化串口、输出日志、显示 VGA 文本、hlt 循环。
  - 学习点：早期内核为何常用“输出 + 死循环”做首个里程碑。

- [kernel/arch/x86_64/serial.c](../kernel/arch/x86_64/serial.c)
  - 关键函数：serial_init、serial_write。
  - 作用：初始化 COM1 并轮询发送字符。
  - 学习点：UART 初始化顺序、轮询发送模型。

- [kernel/arch/x86_64/vga.c](../kernel/arch/x86_64/vga.c)
  - 关键函数：vga_clear、vga_write。
  - 作用：写入 0xB8000 文本显存。
  - 学习点：字符与颜色属性打包、换行逻辑。

## 2.4 构建与调试入口

- [Makefile](../Makefile)
  - 关键目标：all、iso、run、run-serial、debug、gdb。
  - 作用：统一构建、打包、运行、调试流程。
  - 学习点：freestanding 编译参数、-s -S 调试模式含义。

## 3. 建议断点与观察点

## 3.1 GDB 断点顺序

1. 在 _start 处下断点，确认进入内核入口。
2. 在 setup_page_tables 处下断点，检查页表是否填充。
3. 在 enable_long_mode 处下断点，单步观察控制寄存器写入。
4. 在 long_mode_entry 处下断点，确认已进入 64 位。
5. 在 kmain 处下断点，观察 C 环境是否可用。

## 3.2 关键寄存器检查建议

- 进入 long mode 前后重点观察：CR0、CR3、CR4。
- MSR EFER（0xC0000080）应设置 LME 位。
- 进入 long_mode_entry 后，RSP 应指向 stack_top。

## 4. 现象与代码对应关系

- 终端出现 [serial] LocxOS booting...
  - 对应代码：serial_init 与 serial_write 调用链。

- QEMU 屏幕显示 LocxOS boot ok
  - 对应代码：vga_clear 与 vga_write 调用链。

- 系统停在不再前进
  - 对应代码：kmain 中 hlt 无限循环。
  - 说明：这是当前阶段的预期行为，不是崩溃。

## 5. 学习方法建议

1. 先跑通一次 run-serial，建立“现象 -> 代码”直觉。
2. 再跑 debug + gdb，从 _start 到 kmain 逐步单步。
3. 每次只改动一个点并复测，避免把启动问题叠加。
4. 记录每个关键寄存器在断点处的值，形成自己的启动笔记。
